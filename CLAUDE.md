# Artemis 2 3D Space Simulator — Claude Instructions

## Project Overview
Interactive web-based 3D simulator of the NASA Artemis 2 mission. Shows Earth, Moon, and Orion spacecraft with real orbital mechanics, time control, labels, and trail effects.

## Technology Stack
- **Frontend**: Pure HTML5 + CSS3 + JavaScript (ES modules, no bundler)
- **3D Engine**: Three.js r170 via CDN (esm.sh)
- **Data**: Pre-computed trajectory JSON (generated by Python/skyfield)
- **Hosting**: GitHub Pages (auto-deploy via GitHub Actions)

## Architecture
```
index.html          ← app shell, imports ES modules
src/js/
  main.js           ← Three.js scene setup, render loop
  simulation.js     ← time engine, orbital mechanics, position interpolation
  objects.js        ← Earth/Moon/Artemis mesh creation and texture loading
  labels.js         ← CSS2DRenderer label management
  trails.js         ← ring buffer trail rendering
  controls.js       ← UI panel event handlers
src/css/
  style.css         ← UI overlay styles (glassmorphism dark theme)
assets/textures/    ← Earth/Moon textures (loaded at runtime via CDN fallback)
data/
  artemis2_trajectory.json  ← pre-computed spacecraft positions (1-min resolution)
scripts/
  generate_trajectory.py    ← Python: generates trajectory JSON via skyfield
```

## Coordinate System
- Origin: Earth center
- 1 unit = 1,000 km
- Earth radius: 6.371 units
- Moon radius: 1.737 units
- Earth–Moon average distance: 384.4 units

## Running Locally
```bash
# Serve (needed for ES module imports from file://)
python3 -m http.server 8080
# Open http://localhost:8080
```

## Regenerating Trajectory Data
```bash
pip install -r requirements.txt
python3 scripts/generate_trajectory.py
# Outputs: data/artemis2_trajectory.json
```

## Key Conventions
- All code and comments in English
- No build step — CDN imports only, no npm/node
- Texture fallback: if NASA textures fail to load, use procedural colors
- Time is stored as Unix timestamp (seconds since epoch) internally
- Sim time advances: `simTime += deltaSeconds * speedMultiplier` per frame

## Coding Standards
- ES2022 modules, no transpilation
- Prefer `const` / `let`, arrow functions, destructuring
- Three.js objects disposed properly on scene teardown
- No external state outside module exports
